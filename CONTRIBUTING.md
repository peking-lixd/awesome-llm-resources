# 🤝 贡献指南

感谢你对 Awesome LLM Resources 的兴趣！欢迎贡献！

---

## 📋 贡献类型

### 1. 论文推荐
- 推荐值得收录的经典或前沿论文
- 提供准确的 arXiv 链接
- 撰写简洁的价值说明 (1-2 句)

### 2. 项目推荐
- 推荐优质的开源项目
- 确保项目活跃维护 (最近 3 个月有更新)
- 提供 GitHub Stars 和简介

### 3. 内容完善
- 修正错误信息
- 补充缺失内容
- 改进文档表述

### 4. 功能增强
- 添加新的分类
- 改进页面结构
- 优化视觉效果

---

## 🚀 提交流程

### Step 1: Fork 项目
```bash
# 在 GitHub 上点击 Fork 按钮
```

### Step 2: 克隆仓库
```bash
git clone https://github.com/YOUR_USERNAME/awesome-llm-resources.git
cd awesome-llm-resources
```

### Step 3: 创建分支
```bash
git checkout -b feature/add-paper-xxx
# 或
git checkout -b fix/typo-readme
```

### Step 4: 修改内容
- 在对应目录添加/修改内容
- 遵循现有格式
- 确保链接有效

### Step 5: 提交更改
```bash
git add .
git commit -m "feat: add paper about xxx"
# 或
git commit -m "fix: correct typo in README"
```

### Step 6: 推送并创建 PR
```bash
git push origin feature/add-paper-xxx
# 然后在 GitHub 创建 Pull Request
```

## 📝 格式规范

### 论文格式
```markdown
| # | 论文 | 年份 | 机构 | 引用 | 链接 | 简介 |
|---|------|------|------|------|------|------|
| X | **论文标题** | 202X | 机构 | Xk+ | [📄 arXiv](链接) | 一句话简介 |
```

### 项目格式
```markdown
| 项目 | Stars | 简介 | 链接 |
|------|-------|------|------|
| **项目名** | Xk+ | 简短描述 | [GitHub](链接) |
```

### Commit 信息规范
```
feat: 新功能
fix: 修复问题
docs: 文档更新
style: 格式调整
refactor: 重构
test: 测试相关
chore: 其他
```

---

## ✅ 审核标准

提交的内容会通过以下标准审核：

- ✅ **准确性** - 信息正确，链接有效
- ✅ **相关性** - 与 LLM 领域相关
- ✅ **质量** - 内容有价值，表述清晰
- ✅ **格式** - 遵循现有格式规范
- ✅ **原创** - 非抄袭内容

## 💡 常见问题

### Q: 如何推荐论文？
A: 在 `papers/classic.md` 或 `papers/hot_this_month.md` 中添加，确保是高质量论文。

### Q: 项目需要多少 Stars？
A: 建议 1k+ Stars，但特别优秀的新项目也可收录。

### Q: 多久能合并？
A: 通常 1-3 天内审核，复杂 PR 可能需要更久。

### Q: 可以推荐自己的项目吗？
A: 可以，但请确保质量达标，避免过度自我推广。
